# 自动发布视频工具

本工具运行在 Windows 本机，用于辅助发布视频到抖音、小红书、快手和 B站。

它会自动上传视频、填写标题/正文/话题并上传封面。最终的“发布”按钮由用户检查后手动点击。

## 一、运行要求

- Windows 10 或 Windows 11
- Microsoft Edge（推荐）或 Google Chrome
- 完整交付版已内置 Node.js，无需用户安装运行环境
- 可正常访问各视频平台创作者中心

精简源码版才需要单独安装 Node.js 20 或更高版本。

## 二、最简单的启动方式

1. 解压完整程序文件夹，不要只复制单个 `.bat` 文件。
2. 双击 **`启动工具.bat`**。
3. 完整交付版会直接启动，不需要联网安装依赖。
4. 浏览器会自动打开：

   `http://127.0.0.1:8787`

5. 使用结束后，双击 **`停止工具.bat`**。

如果网页没有自动打开，可以手动在 Edge 或 Chrome 中输入上述地址。

## 三、首次使用

1. 把视频放入 `data/inbox` 文件夹。
2. 横屏和竖屏封面也可以放入 `data/inbox`，工具会自动识别方向。
3. 打开工具网页，点击“扫描”。
4. 检查或修改每个平台的标题、正文和话题。
5. 点击对应平台的“打开发布”。
6. 首次使用某个平台时，在自动打开的浏览器中完成登录。
7. 检查视频、文案和封面，确认无误后手动点击平台的最终发布按钮。

点击工具中的“归档”后，系统会先同步扫描封面，再将选中视频及其位于 `data/inbox` 中的关联横屏、竖屏封面一起移动到 `data/archive/YYYY-MM-DD_HH-mm-ss/`。每次归档使用独立的时间戳目录；同一秒内重复归档会自动追加序号。

## 四、封面要求

- 横屏封面：图片宽度大于或等于高度。
- 竖屏封面：图片高度大于宽度。
- 支持 JPG、JPEG、PNG、WEBP。
- 如果自动扫描结果不正确，可以在网页中分别手动上传横图和竖图。

### 自动匹配规则

扫描封面时，只会匹配当前仍位于 `data/inbox` 中的视频，不会匹配已经归档的视频。

1. 如果 `data/inbox` 中只有一个视频，扫描到的图片会直接关联到该视频，不要求文件名相同。
2. 如果有多个视频，工具会先处理文件名：
   - 忽略扩展名和英文字母大小写。
   - 忽略空格、点号、下划线、横线、括号等分隔符。
   - 从封面名末尾移除 `封面`、`封面图`、`cover`，以及可选的 `横屏`、`竖屏`、`横版`、`竖版`、`landscape`、`portrait`、`视频`。
3. 处理后的封面名和视频名按以下优先级匹配：
   - 名称完全相同，优先级最高。
   - 一个名称是另一个名称的开头，且共同部分至少有 4 个字符。
   - 两者开头至少有 4 个相同字符，并且共同部分占封面名或视频名的 60% 以上。
4. 多个视频得分相同时，为避免误配，该封面不会自动关联，需要调整文件名或在网页中手动上传。
5. 图片宽度大于或等于高度时作为横屏封面；高度大于宽度时作为竖屏封面。
6. 网页手动上传的封面优先级更高，后续扫描不会自动覆盖同方向的手动封面。

推荐让视频和封面使用相同的主题名前缀。例如：

```text
马斯克演讲感悟_科技风字幕版.mp4
马斯克演讲感悟横屏封面.png
马斯克演讲感悟竖屏封面.png
```

视频入库后，工具会自动补扫一次封面；也可以点击网页中的“扫描”重新匹配。若某张自动扫描的封面之前误关联到其他视频，重新匹配成功后会清理旧的自动关联。

## 五、AI 文案配置

AI 文案为可选功能。没有配置 API Key 时，仍可以手动填写文案并使用自动上传功能。

推荐直接在网页的“AI 文案配置”中设置：

1. 选择服务商：OpenAI、DeepSeek、通义千问、Kimi、智谱 GLM，或自定义兼容接口。
2. 检查接口地址和模型名称。
3. 填写用户自己的 API Key。
4. 点击“测试连接”，显示成功后即可生成文案。

服务商升级模型后，可以直接修改“模型”字段，不需要升级本工具。其他提供 OpenAI Chat Completions 兼容接口的服务，可以选择“自定义兼容接口”并填写接口地址和模型名。

配置保存在本机 `data/ai-config.json`。网页只显示脱敏后的密钥，不会返回完整 API Key。交付软件时不要附带开发者自己的 `data` 文件夹、`.env` 或任何真实密钥。

也可以通过 `.env` 提供首次启动默认值：

```env
AI_PROVIDER=openai
AI_BASE_URL=https://api.openai.com/v1
AI_API_KEY=你的_API_Key
AI_MODEL=gpt-5.2
```

旧版 `OPENAI_API_KEY`、`OPENAI_BASE_URL` 和 `OPENAI_MODEL` 配置仍然兼容。

## 六、系统检测和故障处理

网页顶部的“系统检测”会检查：

- 操作系统和 Node.js
- Edge/Chrome 是否安装
- 视频、封面、归档、诊断目录是否可访问
- 各平台适配器版本

每次执行“打开发布”后，工具会生成诊断日志。遇到问题时，可以下载最近的诊断文件并发给技术支持。

常见问题：

### 1. 网页打不开

重新双击 `启动工具.bat`。如果仍打不开，查看：

- `server.err.log`
- `server.out.log`

### 2. 显示端口被占用

先双击 `停止工具.bat`。如果仍提示占用，可以修改 `.env`：

```env
SERVER_PORT=8788
```

之后访问 `http://127.0.0.1:8788`。

### 3. 平台要求重新登录

直接在工具打开的浏览器中登录。登录状态保存在 `data/browser-profiles`。

### 4. 浏览器状态异常

在网页中点击该平台的“重置浏览器”。重置后需要重新登录，操作前会有确认提示。

### 5. 系统文件选择窗口没有关闭

先按 `Esc` 关闭。如果反复出现，请下载最近诊断日志并联系技术支持。

## 七、手动命令

不使用批处理文件时，可以在 PowerShell 中运行：

```powershell
npm install
npm run build
npm run start
```

开发模式：

```powershell
npm run dev
```

正式访问地址是 `http://127.0.0.1:8787`，不是开发模式的 `5173` 端口。

## 八、程序目录

- `data/inbox`：待处理视频和自动扫描的封面
- `data/archive/YYYY-MM-DD_HH-mm-ss`：按归档时间存放的已归档视频和关联封面
- `data/covers`：网页手动上传的封面
- `data/browser-profiles`：各平台登录状态
- `data/diagnostics`：发布诊断日志
- `data/ai-config.json`：网页保存的 AI 服务商配置和 API Key
- `.env`：本机启动默认配置
- `server.out.log`：服务运行日志
- `server.err.log`：错误日志

升级程序时，建议保留 `.env` 和整个 `data` 文件夹。出售或转交给新用户时则应删除开发者自己的 `data` 文件夹，让客户使用自己的账号和 API Key。

## 九、使用边界

- 本工具是发布辅助工具，不会自动点击最终发布按钮。
- 用户应自行检查视频、标题、正文、话题和封面。
- 平台页面升级后可能需要更新适配器。
- 请遵守各平台规则，不要用于垃圾内容、批量骚扰或绕过平台风控。

## 十、给销售和技术支持人员

交付前应确认：

1. `.env.example` 中没有真实 API Key。
2. 不要把开发者自己的 `.env`、登录资料或视频素材交付给客户。
3. 客户电脑已安装 Edge 或 Chrome。
4. 完整交付版中包含 `runtime` 和 `node_modules` 目录。
5. 出现问题时优先收集诊断 JSON、`server.err.log` 和系统检测截图。

当前完整交付版已经内置 Node.js 运行环境。正式大规模销售前，仍建议进一步制作 Windows 安装程序并支持在线更新。
