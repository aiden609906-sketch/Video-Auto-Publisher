# 自动发布视频工具 V3.0

这是一个运行在 Windows 本机的视频发布辅助工具，用于统一管理视频、封面和平台文案，并把准备好的内容填写到抖音、快手和 B站创作者页面。小红书采用辅助手动发布模式。

工具不会自动点击各平台最后的“发布”按钮。自动填写结束后，请人工检查视频、封面、标题、正文、话题和 AI 声明，再决定是否发布。

## 支持的平台和功能

| 平台 | 当前模式 | V3.0 的处理内容 | 最后一步 |
| --- | --- | --- | --- |
| 抖音 | 自动填写 | 上传视频；上传横屏和竖屏封面；填写标题、正文及最多 5 个纯文本话题；选择“内容由AI生成”声明 | 用户检查后手动发布 |
| 快手 | 自动填写 | 上传视频和封面；按“标题、正文、最多 4 个话题”写入作品描述；选择 AI 生成声明 | 用户检查后手动发布 |
| B站 | 自动填写 | 上传视频；完成封面制作窗口中的两种封面；填写标题、正文和标签；选择“含AI生成内容” | 用户检查后手动发布 |
| 小红书 | 辅助手动 | 打开小红书发布页，打开视频和封面所在目录，并把“标题、正文、话题”复制到剪贴板 | 用户手动上传、粘贴、检查并发布 |

平台会不定期调整创作者页面。如果某个平台突然出现漏填、按钮无法点击或封面窗口没有关闭，请先停止自动操作并保留页面现场，再按“故障处理”收集诊断信息。

## 运行要求

- Windows 10 或 Windows 11。
- Microsoft Edge（推荐）或 Google Chrome。
- 能正常访问对应平台的创作者中心。
- 完整交付版应包含内置 Node.js 运行环境和依赖，普通用户不需要单独安装。
- 源码版需要 Node.js 20 或更高版本。

## 快速开始

### 1. 准备素材

把待处理文件放入 `data/inbox`：

```text
data/inbox/
  视频主题.mp4
  视频主题横屏封面.png
  视频主题竖屏封面.png
```

支持的视频格式：MP4、MOV、M4V。

支持的封面格式：JPG、JPEG、PNG、WEBP。

建议视频和封面使用相同的主题名前缀。横屏封面应当宽度大于或等于高度，竖屏封面应当高度大于宽度。

### 2. 启动工具

1. 解压完整程序文件夹，不要只复制单个批处理文件。
2. 双击 `启动工具.bat`。
3. 等待浏览器打开 `http://127.0.0.1:8787`。
4. 如果网页没有自动打开，在 Edge 或 Chrome 中手动访问该地址。

使用结束后双击 `停止工具.bat`。

### 3. 扫描并检查任务

1. 在工具页面点击“扫描”。
2. 选择识别到的视频任务。
3. 检查横屏、竖屏封面是否匹配正确。
4. 检查各平台的发布账号、标题、正文和话题。
5. 可以手动编辑文案，也可以配置 AI 后生成平台文案。

扫描由用户主动触发。文件放入 `data/inbox` 后如果界面没有变化，请点击“扫描”，不要反复重启程序。

### 4. 打开平台发布页

点击目标平台的“打开发布”。

首次使用某个平台或新增账号时，需要在打开的浏览器窗口中登录。不同平台和不同账号使用各自独立的浏览器资料目录，后续会复用登录状态。

抖音、快手和 B站会自动执行填写流程，并在工具中显示每一步的结果。某个关键步骤无法验证时，流程会停下来，不会把“点过按钮”当作成功。

### 5. 人工检查并发布

自动填写结束后，至少检查：

- 视频是否正确且上传完整。
- 封面是否正确应用。
- 标题、正文和话题是否完整。
- AI 生成内容声明是否已经选中。
- 页面是否还有未关闭的弹窗或错误提示。

确认无误后，由用户在平台页面手动点击最终发布按钮。

### 6. 归档素材

发布完成后，可以在工具中点击“归档”。系统会先重新扫描封面，再把当前视频及其位于 `data/inbox` 中的关联封面移动到：

```text
data/archive/YYYY-MM-DD_HH-mm-ss/
```

每次归档使用独立目录；同一秒重复归档时会自动追加序号。网页手动上传到 `data/covers` 的封面不会被错误当作 inbox 原文件移动。

## 小红书辅助手动模式

小红书没有使用网页自动填写，以降低页面自动化触发平台风控的风险。

点击小红书“打开发布”后，工具会：

1. 使用所选账号对应的独立浏览器资料打开小红书发布页。
2. 打开视频和封面所在的文件夹。
3. 把当前小红书文案写入系统剪贴板，格式为：

   ```text
   标题

   正文

   #话题1 #话题2
   ```

随后由用户在小红书页面中选择视频和封面、粘贴文案、补充需要的声明并发布。工具显示“材料已准备”只表示页面、目录和剪贴板已经准备好，不表示视频已经上传或笔记已经发布。

## 封面自动匹配

封面扫描只匹配当前仍位于 `data/inbox` 中的视频，不匹配已经归档的视频。

匹配规则：

1. 忽略扩展名、英文字母大小写、空格、点号、下划线、横线和常见括号。
2. 从封面文件名末尾移除 `封面`、`封面图`、`cover`，以及可选的 `横屏`、`竖屏`、`横版`、`竖版`、`landscape`、`portrait`、`视频`。
3. 处理后名称完全相同的优先级最高。
4. 名称互为前缀时，共同部分至少需要 4 个字符。
5. 其他情况要求开头至少 4 个字符相同，且共同部分占封面名或视频名的 60% 以上。
6. 多个视频得到相同最高分时不会自动猜测，需要调整文件名或在页面中手动指定封面。
7. 网页手动上传的封面优先级高于自动扫描结果，后续扫描不会覆盖同方向的手动封面。

## 账号和浏览器状态

- 每个平台可以配置多个发布账号。
- 每个账号使用独立的浏览器资料目录，保存在 `data/browser-profiles/accounts/`。
- 自动填写同一平台同一账号时，不允许两个任务同时运行。
- 程序会尽量复用同一账号已经打开的发布页，并关闭重复的空白页或同平台页面。
- “重置浏览器”会删除对应账号的本地登录资料，之后需要重新登录。

重置操作不可恢复。只有在登录状态或浏览器资料确实异常时才使用。

## AI 文案配置

AI 文案是可选功能。没有配置 API Key 时，仍可手动填写文案并使用发布辅助功能。

建议在工具页面的“AI 文案配置”中完成设置：

1. 选择 OpenAI、DeepSeek、通义千问、Kimi、智谱 GLM 或自定义兼容接口。
2. 检查接口地址和模型名称。
3. 填写自己的 API Key。
4. 点击“测试连接”，成功后再生成文案。

配置保存在本机 `data/ai-config.json`。页面只显示脱敏后的密钥。不要把自己的 `.env`、`data/ai-config.json` 或完整 `data` 目录交给其他用户。

源码版也可以在 `.env` 中提供默认值：

```env
AI_PROVIDER=openai
AI_BASE_URL=https://api.openai.com/v1
AI_API_KEY=你的_API_Key
AI_MODEL=gpt-5.2
```

## 常用目录

| 路径 | 用途 |
| --- | --- |
| `data/inbox` | 待扫描的视频和封面 |
| `data/archive` | 已归档的视频及关联封面 |
| `data/covers` | 在工具页面中手动上传的封面 |
| `data/browser-profiles` | 各平台、各账号的登录资料 |
| `data/diagnostics` | 自动填写诊断文件和调试截图 |
| `data/ai-config.json` | AI 服务商配置和 API Key |
| `.env` | 本机启动配置 |
| `server.out.log` | 后台服务运行日志 |
| `server.err.log` | 后台服务错误日志 |

## 故障处理

### 网页打不开

1. 双击 `停止工具.bat`。
2. 再双击 `启动工具.bat`。
3. 检查 `server.err.log` 和 `server.out.log`。
4. 手动访问 `http://127.0.0.1:8787/api/health`；正常时应返回 `ok`。

### 端口被占用

先停止旧程序。如果 8787 端口必须保留给其他程序，可以修改 `.env`：

```env
SERVER_PORT=8788
```

之后访问 `http://127.0.0.1:8788`。

### 平台要求重新登录

直接在工具打开的浏览器中登录，然后回到工具重新点击“打开发布”。不要把验证码、Cookie 或登录资料发送给他人。

### 自动填写停在某一步

不要反复点击页面。先记录工具中显示的失败步骤，并保存：

- 当前平台页面截图。
- 最近的诊断文件。
- `server.err.log`。
- 系统检测页面截图。

自动填写显示“失败”不一定代表前面所有内容都没填。例如封面已经可见，但封面完成状态无法验证时，工具仍会如实报告该步骤未完成。

### 系统文件选择窗口没有关闭

按 `Esc` 关闭。如果重复出现，停止当前流程并收集诊断信息。

### 浏览器状态异常

确认选中了正确账号，再使用该账号旁的“重置浏览器”。重置后必须重新登录。

## 升级和 V3.0 恢复

升级程序前，备份并保留：

- `.env`
- 整个 `data` 目录

当前稳定恢复点为 Git 标签 `V3.0`，对应提交 `227fe9a`。源码维护者可以安全地从标签创建恢复分支：

```powershell
git switch -c restore-v3 V3.0
```

不要直接执行 `git reset --hard`，否则可能覆盖尚未提交的本地修改。

## 使用边界和注意事项

- 本工具是发布辅助工具，不是无人值守批量发布系统。
- 最终发布必须由用户检查后手动完成。
- 小红书只准备页面、素材目录和剪贴板，不自动操作发布表单。
- 请遵守各平台规则，不要用于垃圾内容、批量骚扰或规避平台风控。
- 平台页面、按钮文案、弹窗结构或上传流程变化后，适配器可能需要更新。
- 登录资料、API Key、视频素材和诊断截图都可能包含敏感信息，不要提交到公开仓库或发给无关人员。

## 交付、销售和售后检查清单

交付前确认：

1. 客户电脑为 Windows 10/11，并安装 Edge 或 Chrome。
2. 完整交付包包含 `runtime`、`node_modules`、已构建的 `dist` 和全部批处理文件。
3. `.env.example` 不包含真实 API Key。
4. 不交付开发者自己的 `.env`、`data`、浏览器登录资料、视频素材或诊断截图。
5. 新客户首次启动时使用自己的账号登录，并配置自己的 AI 服务。
6. 至少演示一次“放入素材、扫描、检查文案、打开发布、人工确认、归档”的完整流程。
7. 明确告知客户：平台页面更新可能需要软件升级，不能保证外部页面永久不变。

售后排查优先收集：

- 软件版本或 Git 标签。
- 失败平台和失败步骤。
- 工具生成的诊断文件。
- 已脱敏的平台页面截图。
- `server.err.log` 和系统检测结果。

不要要求客户发送 API Key、Cookie、短信验证码或整个浏览器资料目录。

## 源码运行与验证

源码版安装和启动：

```powershell
npm install
npm run build
npm run start
```

开发模式：

```powershell
npm run dev
```

类型检查和全量测试：

```powershell
npm run check
node --import tsx --test tests/*.test.ts
```

正式服务地址由 `SERVER_PORT` 决定，默认是 `http://127.0.0.1:8787`；Vite 开发页面使用 5173 端口，不要混淆。
