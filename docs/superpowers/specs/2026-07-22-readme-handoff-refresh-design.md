# README 与 AI 交接文档更新设计

日期：2026-07-22

## 目标

更新项目的两份入口文档，使它们与当前 `V3.0` 稳定版本一致，并明确区分读者：

- `README.md` 面向普通 Windows 使用者，同时在末尾覆盖软件交付、销售和售后支持人员需要的信息。
- `AI_HANDOFF.md` 面向接手维护的开发者或 AI，说明当前代码结构、发布工作流、耦合点、风险和验证方法。

## README 设计

README 采用“用户任务优先、交付信息后置”的结构，避免在主要操作流程中穿插内部实现细节。

主要内容：

1. 工具定位、支持平台和“最终发布由用户确认”的使用边界。
2. V3.0 平台能力对照：抖音、快手、B站为托管式自动填写；小红书为辅助手动发布。
3. 素材准备、封面命名、启动、首次登录和标准发布步骤。
4. 小红书辅助模式的准确行为：打开发布页面和视频目录，自动把标题、正文、话题复制到剪贴板，页面上传、粘贴和最终发布由用户完成。
5. AI 文案配置、素材归档、常见问题、升级和从 `V3.0` 标签恢复。
6. 交付、销售和售后检查清单。
7. 源码运行和验证命令作为末尾附录。

README 不记录私有实现方法名、DOM 选择器或历史调试过程。

## AI_HANDOFF 设计

AI_HANDOFF 以当前代码为准，不再沿用旧的 V2.0.5 状态记录。主要内容：

1. 稳定基线：`main`、`V3.0` 和提交 `227fe9a`。
2. 技术栈、目录结构、运行命令和敏感数据边界。
3. 从前端请求到 Express、Store、Publisher、V3 workflow、平台页面和结果持久化的完整数据流。
4. `src/server/publish/` 中的工作流接口、阶段结果、账号锁、页面所有权和抖音适配器。
5. `src/server/publisher.ts` 中仍保留的浏览器管理以及抖音、快手、B站兼容实现。
6. 四个平台当前发布模式、字段处理规则和流程差异。
7. 主要耦合点及影响范围，包括大型 `publisher.ts`、集中式 `index.ts`、外部 DOM/中文文案/时序、V3 与旧布尔结果映射、浏览器上下文和账号复用。
8. 修改功能时的文件导航、测试映射、真实页面验证原则和禁止自动点击最终发布的边界。
9. 已知风险、建议重构顺序、接手步骤和版本恢复方法。

## 准确性原则

- 文档内容以当前代码和 `V3.0` 标签为准，不沿用已经失效的测试数量和平台步骤。
- README 只承诺程序能够验证的能力，不把“已经点击”写成“已经成功”。
- 小红书明确标记为辅助手动模式，不描述为自动上传。
- 不读取、复制或记录 `.env`、`data/ai-config.json`、浏览器登录资料和真实账号内容。
- 本次只修改文档，不调整运行逻辑。
